### Decomposition和思路的对比。

**Decomposition:**

​	$ (time,day,week)$这是文件中提到的时间特征，表示为temporal feature。

​	基于相似度关系$\rightarrow$获得空间特征，表示为spatio feature。

​					Spatiotemporal Feature  $\xLeftrightarrow{MAP}$  Normal Data

​					True Data || Normal Data $\rightarrow$ 差值数据，进而异常判断。

​	说到底，这个文章的思路就是，异常事件会立刻影响交通数据但不会立刻响应到时空特征上，因此时空特征代表的就是正常状态的交通数据。

**思路**

​				True Data $\xrightarrow{mining}$ Spatiotemporal Feature$\xrightarrow{restore}$ Generated Data

​	简单来说，我们挖掘True Data中的时空特征，而后利用Spatiotemporal Feature中包含的信息还原Data。真实情况下True Data中不会包含异常数据，那么当异常数据输入时，模型没有见过所以就会导致时空信息挖掘的不正确，因此就会获得不好的Spatiotemporal Feature，因此就难以还原。最后我们利用还原出的数据和真实数据的差值识别异常。



DCdetector关注的难点

​	数据中包含正常数据和异常数据，正常数据占据大多数，因此，构建重建模型可以学习到正常数据的特征空间，通过这个空间中的向量我们可以还原正常数据但是很难还原异常数据。我们采用对比学习的方式进行学习，因此不需要有标签数据也不需要先验知识因此，我们可以识别从未出现异常。





**基于正负对的对比学习**

​		Data Point $\xrightarrow{AUG}$ Changed Data Point$\xrightarrow{ Pre-training \ model}$ Feature (High-dim Feature Space)

​	而后采用生成的特征，用于下游任务。
